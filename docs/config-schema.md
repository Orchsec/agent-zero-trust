# Config Schema

Configs may be YAML or JSON. Required top-level section:

- `agent`

Optional sections use safe defaults:

- `project`
- `tools`
- `identity`
- `authentication`
- `authorization`
- `logging`
- `traceability`
- `memory`
- `rag`
- `sandboxing`
- `input_output_controls`
- `recovery`
- `governance`
- `supply_chain`
